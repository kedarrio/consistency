# Prompt 1.3 — Component Library

> Copy and paste this prompt to an AI agent to complete Phase 1.3.

---

## Task

Build all reusable UI components for the Consistency app.

## Context

- Design system tokens in `project/design/tokens.json` (already set up as CSS variables)
- All component specs in `project/design/components/`
- Reference images in `project/references/` (HabitCard.png, Toggle.png, SegmentedPicker.png)
- Rules in `project/design/claude-guardrails.md`

## Components to Build

### Priority 1 (Core)
1. `Button` — primary, secondary, icon, FAB, danger variants
   Spec: `project/design/components/buttons.md`

2. `HabitCard` — dashboard card with all 4 quick-action variants
   Spec: `project/design/components/cards.md`
   Reference: `project/references/HabitCard.png` — match exactly

3. `Toggle` — iOS-style toggle switch
   Spec: `project/design/components/inputs.md`
   Reference: `project/references/Toggle.png` — match exactly

4. `SegmentedPicker` — Apple-style segmented control
   Spec: `project/design/components/inputs.md`
   Reference: `project/references/SegmentedPicker.png` — match exactly

### Priority 2 (Inputs)
5. `TextInput`, `TextArea`, `NumberInput`
6. `DatePicker`, `TimePicker` (Apple-style scrolling wheel modal)
7. `Slider`

### Priority 3 (Progress & Charts)
8. `ProgressBar` (mini 6px, large 10px)
9. `CircularProgress` (SVG-based timer ring)
10. `BarChart` (7-day, using Recharts)
11. `LineChart` (using Recharts)

### Priority 4 (Navigation & Overlays)
12. `TopNav` — contextual top navigation bar
13. `BottomNav` — 4-item fixed bottom navigation
14. `HamburgerDrawer` — right slide-in drawer with habit links
15. `Modal` — base centered modal
16. `BottomSheet` — slide-up modal variant

## Acceptance Criteria

- [ ] All components match their spec files exactly
- [ ] HabitCard matches `HabitCard.png` reference image
- [ ] Toggle matches `Toggle.png` reference image
- [ ] SegmentedPicker matches `SegmentedPicker.png` reference image
- [ ] All components use CSS variables from tokens.css (no hardcoded values)
- [ ] All touch targets ≥ 44px
- [ ] Dark text (`#2A2A2A`) on all habit card surfaces (not white)
